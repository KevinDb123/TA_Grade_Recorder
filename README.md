# TA_Grade_Recorder

# 中科大学生名单作业/实验成绩登记工具

一个基于 Flask 和 openpyxl 的本地成绩登记工具，用来维护中科大学生名单 Excel 中的作业、实验或平时成绩。

它的目标不是重新设计成绩表，而是在尽量保留原始 `xlsx` 结构的前提下，直接对现有名单进行录入、查询和批量更新。

## 功能概览

- 支持读取当前目录下的 `.xlsx` 文件
- 支持导入新的 `.xlsx` 文件并切换当前工作簿
- 自动识别“平时成绩”所在列区间
- 支持不同数量的作业/实验/次数列
- 支持新增、删除、重命名成绩列
- 单人录分
- 批量录分
- 批量录分预览
- 支持“覆盖已有成绩”或“跳过已有成绩”
- 成绩查看页支持按次数查看和按学号/姓名筛选
- 成绩查看页固定每页显示 5 人

## 适用场景

这个项目适合下面这类表格：

- 学生名单保存在 Excel 中
- 学号列固定在第 2 列
- 姓名列固定在第 3 列
- 第 6 行包含“平时成绩”总表头
- 第 7 行是各次作业、实验或平时成绩子表头
- 第 8 行开始是学生数据
- “平时成绩”下方的列数可能变化

也就是说，这个工具重点解决的是：

- 同一门课不同学期的成绩列数不一致
- 需要临时增加或删除一次作业/实验
- 希望直接在原始名单上登记，而不是手工改列和复制样式

## 桌面版打包

项目已经接入 Tauri 桌面壳，发布时可以打包成 Windows `exe` 安装包。

先安装 Node 依赖：

```bash
npm install
```

然后执行正式打包：

```bash
npm run desktop:build
```

生成的安装包默认位于：

```text
src-tauri/target/release/bundle/nsis/TA Grade Recorder_0.1.0_x64-setup.exe
```

如果只想验证桌面壳是否能正常编译，也可以先跑调试构建：

```bash
npx tauri build --debug
```

桌面版的工作方式是：

- Tauri 负责桌面窗口
- 内部会启动本地 Python 后端
- 用户自己选择或导入 `xlsx`
- 安装包里不包含任何真实学生名单文件

## 使用说明

### 1. 录分页面

录分页面主要负责单人录分和批量录分。

你可以在这里完成：

- 切换当前 Excel 文件
- 导入新的 Excel 文件
- 选择当前成绩列
- 重命名当前列
- 新增一列
- 删除当前列
- 按学号或姓名搜索学生
- 保存单个学生的成绩
- 批量粘贴名单并保存成绩

### 2. 成绩列管理

系统会把“平时成绩”下的每一列视为一个可操作的成绩列。

默认情况下，如果某一列没有名称，会自动按顺序显示为：

```text
第1次、第2次、第3次……
```

你也可以手动改成更具体的名字，例如：

- 作业1
- 实验2
- 课堂测验
- 期中补交

### 3. 单人录分

在搜索框中输入：

- 学号
- 姓名

选中学生后，在“本次成绩”中输入分数并保存即可。

支持回车直接保存。

### 4. 批量录分

批量录分支持两种常见格式。

格式一：每行一条“学号 成绩”

```text
21007001 95
21007002 88
21007003 91
```

格式二：只粘贴名单，再统一填写一个成绩

```text
21007001
21007002
21007003
```

然后在“统一成绩”输入同一个分数即可。

### 5. 批量预览

正式保存前，页面会先预览：

- 总条数
- 已匹配人数
- 将写入人数
- 未匹配人数
- 覆盖人数
- 补空人数
- 跳过人数

同时会显示每一行的匹配情况，方便先检查再保存。

### 6. 成绩查看页

成绩查看页可以：

- 查看某一次成绩
- 查看已评分人数
- 查看未评分人数
- 按学号或姓名筛选
- 分页浏览名单

目前分页固定为每页 5 人。

## 技术实现

- 后端：Flask
- Excel 读写：openpyxl
- 前端：原生 HTML / CSS / JavaScript

## 设计原则

这个项目的核心思路是：

- 不改变用户原本的 Excel 使用习惯
- 直接围绕已有名单表格工作
- 尽量保留原始表格样式
- 降低手工维护列结构和录分的成本

## 注意事项

- 录分前请关闭正在占用该 Excel 的 Excel 或 WPS，否则可能无法写入
- 当前版本默认操作项目根目录下的 `.xlsx` 文件
- 该工具面向本地使用，不包含账号系统和多人协作控制
- 为避免误操作，建议先保留一份原始名单副本
